总结一下这段时间在 Vibe Coding 上的实战经验，供大家参考：

1. 需求聊透再动手： 避免返工。每个页面、功能、设计风格都要和 AI 充分讨论，让 AI 补全细节。

2. 锁定热门技术栈： 推荐 Next.js + Supabase + TailwindCSS 等。因为热门技术栈在 AI 模型中训练数据最足，回答的准确度和代码质量是最高的。

3. 接口先行： 每一个接口都要写独立的 API 文档，接入代码前先用 AI 生成 cURL 测试，排除第三方不可控因素。

4. 分批执行 MVP： 任务拆解，先完成最小可行性版本。建议先 Mock 数据搞定前端，再逐个接入 API。

5. 善用开源库： 动手前先问 AI 有没有合适的轮子，和 AI 讨论并确认开源库方案后，再在此基础上修改，大幅减少代码生成量和调试成本，也可以搜索下github。

6. 版本管理意识： 完成一个独立功能就 Git Commit，方便随时回滚，大胆实验。

7. 精准复现 Bug： 报错信息给全（页面、控制台、Network），清晰描述“正确逻辑 vs 错误现象”，无法解决时及时回滚换思路。

8. 投喂充足上下文： 不要只发文字，多发截图（UI参考）、文档（API字典）、示例代码。

9. 引导式修复： 让 AI 像人一样梳理逻辑，在关键步骤输出日志排查。

10. 控制 Token 上限： 每个独立任务新开一个标签页，保持上下文纯净。
11. 尽量多用几个模型，鬼知道相同的描述下，不同的模型会产出什么不同的结果，我目前是 Trae SOLO + Codex  + Claude Code + Gemini 的多重组合，万利且只有一弊。

12. 一定要学会多线 ，一定要用 git worktree 或者 AI 的各种云能力，这很可能是未来区分工程师好坏的重要指标之一。另外，钱都付了，每天都要让丫筋疲力尽。

13. 遇到问题，试着换种语言 prompt，我通常纯英文 prompt，因为我中英文打字太慢。昨天遇到问题卡很久，换成中文反而有思路了。
14. 无论什么 AI 工具，都要让先 Plan，节约乃美好品德。富哥富姐可以忽略。

15. 复杂的问题，可以多给调研资料，让其对比，让其分析，PUA 丫的，不然干活不彻底，总偷懒。

16. AI 头铁，你不头铁，一条路走不通，换条路。

17. 一个 AI 工具的 Token 快用完了，赶紧让其把当前进度，遇到的问题，未来的思路总结成文档，下一个工具好继续，这在人类社会叫交接。

18. 别怀疑你自己，简单的任务，可以自己上，AI 不一定有你快。

19. AI 频繁错误的时候，可以对他草泥马，合理发泄情绪，保持正常心态是我等牛马的重要身心修为。

20. 草泥马不够用的时候，睡觉去，不然键盘不够你砸。


可以通过 Git 回退到 AI 修改之前的状态。以下是几种方法:

```shell
# 如果你还没有 commit 这些改动 丢弃所有修改的文件
git checkout -- .

# 删除新增的文件（Sidebar.tsx）
git clean -fd
```

```shell
# 只回退特定文件
git checkout -- App.tsx
git checkout -- index.html
rm components/Sidebar.tsx
```

```shell
# 回退到上一个 commit
git reset --hard HEAD~1
```
